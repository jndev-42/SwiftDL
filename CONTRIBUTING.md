# Contribution à StremioDownloader

Nous sommes ravis que vous souhaitiez contribuer à **StremioDownloader** ! Voici quelques instructions pour vous aider à démarrer.

## Comment contribuer

### Rapports de bugs
- Vérifiez d'abord si le bug a déjà été signalé dans les *Issues*.
- Si ce n'est pas le cas, ouvrez une nouvelle issue en incluant les étapes pour reproduire le bug, votre version de macOS et le lien de téléchargement qui a échoué.

### Proposer des fonctionnalités
- Les suggestions pour améliorer la gestion des téléchargements ou ajouter des sources sont les bienvenues.
- Ouvrez une issue pour discuter de la fonctionnalité avant de soumettre une Pull Request.

### Pull Requests (PR)
1. Forkez le projet.
2. Créez une branche pour votre fonctionnalité ou correction (`git checkout -b feature/ma-super-fonctionnalite`).
3. Soumettez votre code en respectant le style du projet (SwiftLint est recommandé).
4. Assurez-vous que l'application compile et se lance toujours correctement.
5. Ouvrez une Pull Request avec une description claire de vos changements.

## Style de code
- Suivez les standards de programmation Swift.
- Utilisez la documentation DocC (`///`) pour toutes les méthodes publiques.
- Priorisez l'utilisation de `async/await` pour tout ce qui touche au réseau ou au système de fichiers.
